# "动起来"（MoveOn）移动端 V1.0 产品需求文档

> 使用 UseCase 需求分析方法编写
> 桌面版 PRD：[2026-07-17-moveon-v1-prd.md](2026-07-17-moveon-v1-prd.md)
> 本文档为移动端补充章节，与桌面版共同构成完整产品需求。
> 桌面版已定义的公共业务规则（用户注册规则、密码策略、模组限制等）不再重复，仅标注差异。

---

## 产品概述

- **产品名称**：动起来（MoveOn）
- **产品定位**：面向有健身需求人群的个人运动应用
- **目标平台**：Android 8.0+（优先适配华为 P70 PRO）
- **屏幕适配**：竖屏为主布局，视频播放页支持横竖屏旋转；暂不适配平板
- **技术栈**：Flutter（与桌面版同仓库）
- **V1.0 分发方式**：APK 侧载安装（直接生成 APK 文件，个人使用）
- **V1.0 功能范围**：用户系统 + 视频跟练 + DIY 自定义练习

## SF1′ 应用安装与分发

> 桌面版 SF1（Windows 安装向导/更新/卸载）不适用于移动端，以下为移动端专属。

### SR1 APK 侧载安装

**前置条件**：用户已从可信渠道获取 MoveOn APK 文件

**主场景**：

1. 用户将 APK 文件传输到手机存储（USB 线传或网盘下载）
2. 用户在文件管理器中找到 APK 文件，点击安装
3. 系统弹出"安装未知来源应用"提示
4. 用户前往"设置 → 安全 → 安装未知应用"，允许该文件管理器安装
5. 返回文件管理器，再次点击 APK 文件
6. 系统展示应用安装页面，显示应用名称、所需权限列表（存储读写）
7. 用户点击安装
8. 安装完成，桌面出现 MoveOn 应用图标

**扩展场景**：

- **2a.** 用户通过浏览器下载 APK，下载完成后状态栏弹出通知，用户点击通知直接跳转安装
- **6a.** 用户查看权限列表后可接受或取消安装
- **7a.** 安装过程中存储空间不足（需 ≥ 200MB），系统提示"存储空间不足，请清理后重试"
- **7b.** 安装过程中用户取消，系统停止安装，无残留文件

### SR2 版本更新（V1.0 简化）

**前置条件**：用户已安装旧版本 MoveOn，新版本 APK 已获取

**主场景**：

1. 用户获取新版 APK 文件（与 SR1 渠道相同）
2. 用户点击安装新 APK
3. 系统检测到已有旧版本，提示"是否更新此应用？现有数据将被保留"
4. 用户确认更新
5. 系统覆盖安装，保留本地数据库（用户数据、练习模组、在线视频链接）
6. 安装完成，应用可正常使用

**扩展场景**：

- **3a.** 用户选择取消，保留旧版本，APK 安装终止
- **5a.** 覆盖安装过程中发生错误，系统自动回退至旧版本，用户数据不丢失

## SF2′ 用户系统

> 复用桌面版 SF2 全部业务规则（注册/登录/注销/密码修改）。
> 以下仅标注移动端差异。

| 功能 | 桌面版规则 | 移动端差异 |
|------|-----------|-----------|
| 注册 | 用户名 4-20 位字母/数字/下划线，密码 6-20 位 | **无差异** |
| 登录 | 连续 5 次错误锁定 15 分钟 | **无差异** |
| 注销 | 清除本地登录态，退回游客模式 | **无差异** |
| 密码修改 | 需验证原密码 | **无差异** |
| 登录方式 | 用户名+密码 | **V1.0 仅用户名+密码**（手机验证码登录待后续版本） |

### 移动端扩展场景

**SR1 注册 — 扩展**：

- **移动端：** 注册表单采用单列竖屏布局，输入框宽度占满屏幕，密码输入框提供"显示/隐藏"切换图标
- 软键盘弹出时，表单内容自动上移以避免遮挡

**SR2 登录 — 扩展**：

- **移动端：** 登录页提供"记住我"复选框，勾选后本地保持登录态（SharedPreferences），下次打开应用自动登录
- 桌面版已有"记住我"则复用，未有无需为桌面版补齐

**SR3 用户注销 — 扩展**：

- **移动端：** 注销确认弹窗使用底部 ActionSheet 样式（Material 的 showModalBottomSheet），更符合手机操作习惯

## SF3′ 视频跟练

> 复用桌面版 SF3 全部业务规则。
> 仅新增：移动端 UI 布局适配、视频播放横竖屏旋转、在线视频链接功能直接复用。

### 与桌面版差异总览

| 项目 | 桌面版 | 移动端 |
|------|--------|--------|
| 运动类型网格 | 4 列卡片 Grid | 2 列卡片 Grid（竖屏）/ 4 列（横屏） |
| 视频列表 | 单列宽列表 | 单列列表，卡片占满屏宽 |
| 视频播放 | 固定窗口内播放 | 竖屏半屏播放，**支持旋转横屏全屏** |
| 内置视频 | 第八套广播体操（22MB） | **同桌面版，22MB 打包内置** |
| 在线视频 | 添加/编辑/删除/播放 | **直接复用已有功能**（模型+服务+弹窗） |
| 空状态提示 | 居中文字 | 居中插图 + 文字 |

### SR1 浏览运动类型与视频

> 核心业务流程与桌面版 SR1 一致。

**移动端适配**：

1. 运动类型网格：竖屏 2 列，横屏 4 列
2. 每个卡片展示：分类图标、名称、在线视频数量角标（复用桌面版 FollowHomeScreen 的 GROUP BY 计数逻辑）
3. 点击分类进入视频列表页

**扩展场景**：

- **竖屏 → 横屏旋转**：卡片从 2 列变为 4 列，使用 `OrientationBuilder` 自动响应
- **下拉刷新**：视频列表支持下拉刷新在线视频数据

### SR2 播放跟练视频

> 核心业务流程与桌面版 SR2 一致。

**移动端专属交互**：

1. 用户在视频列表页点击视频
2. 进入视频播放页面
   - **竖屏**：视频播放器占屏幕上半部分（约 40% 高度），下半部分展示视频信息
   - **横屏**：全屏播放，隐藏系统状态栏，仅显示播放控件和返回按钮
3. 视频播放支持：暂停/播放、进度条拖动、音量调节
4. **内置视频**：本地文件播放（`VideoPlayerController.file`）
5. **在线直链视频**：网络流播放（`VideoPlayerController.networkUrl`），加载中显示 Loading 动画
6. **平台链接视频**：弹出确认后调用系统浏览器打开（`url_launcher`，复用桌面版逻辑）
7. 用户点击返回按钮，回到视频列表页

**扩展场景**：

- **2a.** 视频文件缺失或损坏无法播放，系统提示"视频无法播放，请检查安装包是否完整"
- **2b.** 在线直链视频加载失败（网络不通或链接失效），系统提示"请检查视频链接是否有效"
- **3a.** 用户旋转手机至横屏 → 播放器自动进入全屏模式；旋转回竖屏 → 自动退出全屏
- **5a.** 横屏模式下，播放控件 3 秒无操作后自动隐藏，点击屏幕重新显示
- **5b.** 在线直链视频播放时网络切换（WiFi → 移动数据），播放暂停并提示"网络环境已切换，是否继续播放？"

## SF4′ DIY 自定义练习

> 复用桌面版 SF4 全部业务规则（模组上限 10 个、动作时长 5-600 秒、TTS 语音、结束前 5 秒提示音）。

### 与桌面版差异总览

| 项目 | 桌面版 | 移动端 |
|------|--------|--------|
| 模组列表 | 宽屏列表 | 单列卡片列表 |
| 创建/编辑 | 宽屏表单 | 竖屏长表单 + 滚动 |
| 执行页面 | 大屏大字显示 | 手机竖屏显示，横屏可选 |
| TTS 语音 | Windows 系统引擎 | **Android 系统 TTS**（`flutter_tts` 已跨平台） |
| 提示音 | 内置 MP3（`audioplayers`） | **同桌面版，内置 MP3 复用** |
| 拖拽排序 | 长按拖拽 | 长按拖拽（触屏更自然） |

### SR1 创建练习模组（移动端）

> 核心业务规则与桌面版 SR1 一致。

**移动端适配**：

1. 竖屏表单：模组名称 → 运动类型下拉 → 动作列表 → 添加动作按钮 → 保存，单列纵向排列
2. 添加动作：弹出底部 ActionSheet 表单（名称 + 时长），比全屏 Dialog 更符合手机操作习惯
3. 动作列表：每项左滑删除，长按拖拽排序（触屏原生手势）
4. 保存：校验规则同桌面版（名称不为空、至少一个非休息动作）

### SR3 执行练习模组（移动端）

> 核心业务规则与桌面版 SR3 一致。

**移动端专属交互**：

1. 进入执行页面
   - **竖屏**：上半部分大字显示动作名称 + 倒计时秒数，下半部分显示进度条 + 暂停按钮 + 结束按钮
   - **横屏**：全屏大字倒计时（适合放桌面/支架跟练）
2. TTS 语音播报："[动作名称]，时间 [时长] 秒"（调用 Android 系统 TTS 引擎）
3. 倒计时结束前 5 秒播放"铛铛铛"提示音（`audioplayers` 播本地 MP3）
4. 休息动作为倒计时结束前 5 秒不播放提示音
5. 全部完成播放"锻炼结束，好好休息吧"

**扩展场景**：

- **移动端：** 执行期间锁屏或切换到其他应用，倒计时在后台继续运行
- **移动端：** 执行期间来电，倒计时暂停（或有电话接听页面覆盖），通话结束后用户手动继续
- **移动端：** 横屏执行时，翻转手机回竖屏，布局自动切换，倒计时不受影响

## SF5 移动端专项能力

> 桌面版不需要、移动端必须处理的能力。

### SR1 权限管理

**前置条件**：应用首次启动或首次使用需要权限的功能

**主场景**：

1. 应用首次启动，提示需获取"存储读写"权限
2. 系统弹出权限申请对话框
3. 用户点击"允许"
4. 应用正常启动，数据库文件可正常读写

**扩展场景**：

- **3a.** 用户点击"拒绝"，应用进入受限模式：视频播放、在线视频添加等功能可用，但本地数据库操作可能失败；页面顶部显示横幅提示"部分功能需要存储权限"
- **3b.** 用户之后可在系统"设置 → 应用 → MoveOn → 权限"中重新授予权限

**V1.0 需要声明的权限**：

| 权限 | 用途 | Android Manifest |
|------|------|------------------|
| INTERNET | 在线视频流播放、浏览器打开链接 | `<uses-permission>` |
| ACCESS_NETWORK_STATE | 网络状态检测（WiFi/移动数据切换） | `<uses-permission>` |
| READ/WRITE_EXTERNAL_STORAGE | 数据库文件读写 | 仅 Android 10 以下需要 |

### SR2 离线模式

**前置条件**：应用已安装，首次已完成初始化

**主场景**：

1. 用户在无网络环境下打开应用
2. 系统检测到无网络连接（通过 `connectivity` 检测）
3. 以下功能正常使用：
   - 用户登录/注册（本地验证）
   - 内置视频播放（离线可用）
   - DIY 模组创建/管理（本地数据库）
   - 模组练习执行（TTS + 本地音频）
4. 以下功能受限：
   - 在线视频播放 → 点击时提示"当前无网络连接"
   - 平台链接跳转 → 点击时提示"需要网络连接才能打开"

**扩展场景**：

- **2a.** 网络恢复后应用自动切换到在线模式，无需用户手动操作

### SR3 应用启动与后台

**前置条件**：应用已安装

**主场景**：

1. 用户点击桌面 MoveOn 图标
2. 应用展示启动画面（Splash Screen，MoveOn Logo）
3. 启动画面持续至 Flutter 引擎初始化完毕（约 1-2 秒）
4. 自动进入主页（底部 3 Tab 导航，复用桌面版 HomeScreen 结构）

**扩展场景**：

- **1a.** 用户通过最近任务列表切换回 MoveOn，不显示启动画面，直接恢复上次离开的页面状态
- **1b.** 应用被系统从后台回收（内存不足），下次启动走正常启动流程（显示 Splash → 主页）

## 附录：移动端全局设计约定

| 约定项 | 说明 |
|--------|------|
| **导航结构** | 底部 3 Tab：跟练 / DIY / 我的（与桌面版一致） |
| **屏幕方向** | 竖屏为主；视频播放页 + 练习执行页支持横竖屏旋转 |
| **布局方式** | `OrientationBuilder` 响应旋转；竖屏 2 列 / 横屏 4 列 Grid |
| **弹窗风格** | 确认/操作类弹窗使用底部 ActionSheet（`showModalBottomSheet`）；表单/输入类使用全屏 Dialog |
| **运动类型** | 瑜伽、有氧操、跳绳、塑形、体操、普拉提、拉伸、冥想（与桌面版一致） |
| **视频策略** | 内置视频：第八套广播体操（22MB，480p，打包在 APK 中）；其他分类：用户自行添加在线视频链接 |
| **TTS 语音引擎** | Flutter TTS 插件 → Android 系统 TTS 引擎（`flutter_tts` 已跨平台） |
| **数据存储** | 仅本地存储（SQLite，`sqflite` 标准版）；通过 `path_provider` 获取应用数据目录 |
| **注册方式** | 用户名 + 密码（V1.0 不做手机验证码登录） |
| **分发方式** | APK 侧载安装（`flutter build apk --release`）；V1.0 不上架应用商店 |
| **目标版本** | Android 8.0+ (API 26+)，compileSdk 34，minSdk 26，targetSdk 34 |
| **权限** | INTERNET、ACCESS_NETWORK_STATE、存储权限（Android 10 以下） |
