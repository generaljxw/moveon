# 在线视频链接 — 功能设计文档

> 为"视频跟练"模块新增在线视频链接添加与播放能力

---

## 1. 需求概述

V1.0 跟练模块仅支持播放内置视频（第八套广播体操），用户无法自行扩展视频内容。本功能允许用户在各运动分类下添加在线视频链接（收藏夹模式），支持直链播放和平台链接跳转两种方式。

### 核心场景

| 场景 | 描述 |
|------|------|
| **添加在线视频** | 用户在任意分类下添加视频名称 + URL，系统自动判断视频类型（直链/平台链接） |
| **播放直链视频** | 系统使用原生播放器播放 MP4/WebM 直链视频 |
| **打开平台链接** | 系统调用默认浏览器打开 B站、YouTube 等平台链接 |
| **管理收藏视频** | 用户可编辑已添加视频的名称和链接，或删除不再需要的视频 |

---

## 2. 架构设计

### 2.1 数据模型

新增 `online_videos` 表（SQLite），与现有 `exercise_modules` / `exercise_actions` 平级：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 主键 |
| `user_id` | INTEGER | NOT NULL, FK → users.id | 归属用户 |
| `category` | TEXT | NOT NULL | 所属运动分类名（如"瑜伽"） |
| `title` | TEXT | NOT NULL | 视频名称，用户自定义 |
| `url` | TEXT | NOT NULL | 视频链接地址 |
| `video_type` | TEXT | NOT NULL, DEFAULT 'link' | `direct`（直链视频）或 `link`（平台链接） |
| `created_at` | TEXT | NOT NULL | 添加时间 ISO 8601 |

**video_type 自动判断规则：**
- URL 以 `.mp4` / `.webm` / `.mkv` / `.mov` 结尾 → `direct`
- 否则 → `link`
- 用户可在添加/编辑时手动切换

### 2.2 播放分发逻辑

```
用户点击在线视频
  ├─ video_type = "direct" → VideoPlayerController.network(url) → 原生全屏播放
  └─ video_type = "link"   → url_launcher.launchUrl(url) → 系统浏览器打开
```

直链播放复用现有 `VideoPlayerScreen`，通过构造参数区分本地/网络模式。

### 2.3 权限模型

- **用户私有**：在线视频属于添加者，登录后才可见和操作
- **游客模式**：仅展示内置视频，不显示 + 号和在线视频

## 3. UI 交互设计

### 3.1 视频列表页（VideoListScreen）

```
┌─────────────────────────────────┐
│  ← 瑜伽                    [+] │  ← AppBar 右侧 + 号添加
├─────────────────────────────────┤
│ ▶ 内置：柔和瑜伽入门    [本地] │  ← 内置视频（现有样式）
│ 🌐 B站流瑜伽30分钟      [在线] │  ← 在线视频（地球图标 + 绿色标签）
│ 🌐 晨间拉伸直链        [在线] │  ← 直链类型同样显示在线标签
│                                 │
│         ← 左滑删除             │  ← Dismissible 确认删除
│         点击进入编辑            │  ← 点击弹出编辑弹窗
└─────────────────────────────────┘
```

### 3.2 添加/编辑弹窗（AddVideoDialog）

```
┌─────────────────────────────────┐
│        添加在线视频              │
│                                 │
│  视频名称                        │
│  ┌─────────────────────────┐    │
│  │ B站流瑜伽30分钟          │    │
│  └─────────────────────────┘    │
│                                 │
│  视频链接                        │
│  ┌─────────────────────────┐    │
│  │ https://bilibili.com/... │    │
│  └─────────────────────────┘    │
│                                 │
│  类型: ○ 在线链接  ○ 直链视频   │  ← 自动判断 + 可手动切换
│                                 │
│           [取消]  [保存]        │
└─────────────────────────────────┘
```

**输入校验：**
- 名称：必填，1-50 字符，trim 后非空
- URL：必填，必须以 `http://` 或 `https://` 开头
- 同一分类下 URL 不重复（同一用户）

### 3.3 列表项标签

| 视频来源 | 图标 | 标签文字 | 标签色 |
|----------|------|----------|--------|
| 内置视频 | `play_circle_fill` | 本地 | 绿色 |
| 在线直链 | `language`（地球） | 在线 | 蓝色 |
| 平台链接 | `open_in_browser` | 在线 | 橙色 |

### 3.4 在线视频卡片样式

- 左侧图标：地球图标 + 半透明白色圆形背景（与分类卡片图标风格一致）
- 右侧标签：胶囊形 Chip，"在线"文字
- 卡片底色：白色（与内置视频卡片一致）
- 点击行为：直链→播放器跳转，平台链接→确认弹窗"将使用浏览器打开此链接"

## 4. 边界与约束

### 4.1 功能边界

| 项目 | 决策 |
|------|------|
| 单用户在线视频数 | 不限制（与 DIY 模组不同） |
| 视频分类 | 仅限现有 8 种运动分类 |
| 内置视频管理 | 不可编辑、不可删除 |
| 跨用户可见 | 否（私有收藏） |
| 离线缓存 | V1.0 不支持 |
| 播放历史 | V1.0 不记录 |

### 4.2 错误处理

| 场景 | 处理 |
|------|------|
| 直链 URL 失效/不可达 | 原生播放器报错 → 展示"视频无法播放"错误页，引导检查链接 |
| 平台链接格式异常 | url_launcher 打开失败 → Toast 提示"无法打开此链接" |
| 网络未连接 | 播放前不检测，播放时由底层报错 → 错误页提示 |
| 重复添加同一 URL | 保存时校验 → 弹窗提示"该链接已存在于此分类" |

## 6. 技术选型

| 项目 | 选型 | 理由 |
|------|------|------|
| 网络流播放 | video_player_win（现有） | `VideoPlayerController.network()` 已支持 |
| 外部浏览器 | url_launcher | Flutter 官方插件，稳定可靠 |
| 数据持久化 | SQLite（现有 sqflite_common_ffi） | 与现有架构一致 |
| 状态管理 | Provider（现有） | 无需引入新模式 |
| 弹窗表单 | AlertDialog + TextField | 轻量级，无需引入第三方 |
